Возможности

Современный GUI - Интерфейс на Tkinter с прокручиваемым списком файлов и поддержкой колеса мыши

Умная группировка - Автоматическая группировка файлов по типу квантования (Q4_K_M, Q5_K_S и т.д.)

Возобновление скачивания - HTTP Range запросы с защитой от fallback (обработка 200 vs 206)

Параллельная загрузка - Настраиваемое количество потоков (авто-определение ядер CPU)

Проверка хэшей - SHA256 верификация через метаданные HF LFS

Умная склейка - Использует llama-gguf-split из llama.cpp

CLI поддержка - Подкоманды download и merge для автоматизации

Умный поиск - Автоматически находит llama-gguf-split в системе

Сохранение пути - Путь к llama.cpp сохраняется между сессиями

Повторные попытки - Автоматические повторы с экспоненциальной задержкой (3 попытки)

ETA в реальном времени - Скорость (MB/s) и оставшееся время

Многоязычность - Документация на 6 языках

python gguf_merge_tool.py

Использование CLI
Скачать конкретные файлы

python src/gguf_merge_tool.py download -r unsloth/Qwen3.5-122B-A10B-GGUF -d ./models -f Qwen3.5-122B-A10B-Q4_K_M-00001-of-00008.gguf Qwen3.5-122B-A10B-Q4_K_M-00002-of-00008.gguf
Скачать весь репозиторий

python src/gguf_merge_tool.py download -r unsloth/Qwen3.5-122B-A10B-GGUF -d ./models
Скачать с авторизацией (приватный репозиторий)

python src/gguf_merge_tool.py download -r ваш-логин/ваша-модель -d ./models -t hf_xxxxxxxxxxxxxxxxxx
Склеить части

python src/gguf_merge_tool.py merge -d ./models -o ./models/merged_model.gguf -f
Справка по аргументам CLI

Команда: download
-d, --dir - Папка для сохранения файлов - Обязательно: Да
-r, --repo - ID репозитория Hugging Face - Обязательно: Да
-b, --branch - Ветка репозитория (по умолчанию: main) - Обязательно: Нет
-t, --token - Токен доступа HF - Обязательно: Нет
-f, --files - Список файлов для скачивания - Обязательно: Нет

Команда: merge
-d, --dir - Папка с частями - Обязательно: Да
-o, --output - Путь к выходному файлу - Обязательно: Нет
-f, --force - Перезаписывать существующий файл - Обязательно: Нет
Использование GUI
Шаг 1: Загрузка репозитория

Введите ID репозитория Hugging Face (например, unsloth/Qwen3.5-122B-A10B-GGUF)
Введите ветку (по умолчанию: main)
Введите ваш HF токен (требуется для приватных репозиториев)
Нажмите кнопку "Загрузить"
Шаг 2: Выбор файлов

Файлы автоматически группируются по типу квантования
Нажмите "Выбрать всё" для выбора всех файлов
Или выберите отдельные файлы с помощью чекбоксов
Шаг 3: Скачивание

Нажмите "Скачать" или "Скачать всё"
Следите за прогрессом в реальном времени:

    Прогресс-бар показывает проценты

    Строка статуса показывает: [1/4] файл.gguf: 45% | 4.5 MB/s | ETA: 2ч 15м

Шаг 4: Проверка (опционально, но рекомендуется)

Нажмите кнопку "Проверить"
Инструмент проверяет SHA256 хэши по метаданным HF
Показывает сообщение об успехе, если все хэши совпадают
Шаг 5: Склейка (опционально)

Выберите минимум 2 части (чекбоксы)
Нажмите кнопку "Склеить"
Инструмент вызывает llama-gguf-split --merge
Выходной файл сохраняется с правильным базовым именем
Настройка
Путь к llama.cpp

В GUI найдите "Путь к llama.cpp":
Нажмите "Найти" для автоматического поиска
Или нажмите "Обзор" для ручного выбора
Путь сохраняется в llama_path.txt для сохранения между сессиями
Настройка потоков

Измените константу JOBS в коде (по умолчанию: min(4, os.cpu_count() or 2)):
В файле gguf_merge_tool.py
JOBS = min(4, os.cpu_count() or 2) - Измените это значение
Запас места на диске

Измените DISK_SPACE_MARGIN (по умолчанию: 1.1 = 10% запаса):
DISK_SPACE_MARGIN = 1.1
Размер чанка загрузки

Измените CHUNK_SIZE (по умолчанию: 1 МБ):
CHUNK_SIZE = 1024 * 1024

Решение проблем
llama-gguf-split не найден

Скачайте из релизов llama.cpp
Поместите в папку tools/ или укажите путь в GUI
HTTP 401 Unauthorized или HTTP 403 Forbidden

Введите ваш токен Hugging Face
Убедитесь, что токен имеет доступ на чтение к репозиторию
Формат токена: hf_xxxxxxxxxxxxxxxxxx
Недостаточно места на диске

Освободите место на диске (нужно 10% запаса для склейки)
Измените папку загрузки на другой диск
Используйте DISK_SPACE_MARGIN для регулировки запаса
Загрузка зависла на 0%

Проверьте интернет-соединение
Проверьте правильность токена
Проверьте, не приватный ли репозиторий
Проверьте, существует ли репозиторий
Файл испорчен после возобновления

Инструмент имеет встроенную защиту от Range fallback
Если сервер не поддерживает Range (возвращает 200 вместо 206), загрузка начинается с нуля
Это сделано намеренно для предотвращения порчи файлов
GUI не запускается

Проверьте, установлен ли Tkinter (обычно входит в состав Python)
На Linux: sudo apt-get install python3-tk
На macOS: brew install python-tk
Советы по производительности

Много мелких файлов - Оставьте потоки по умолчанию (4)
Мало больших файлов - Увеличьте потоки до 8
Медленная сеть - Уменьшите потоки до 2
Быстрый NVMe SSD - Оставьте по умолчанию или увеличьте
Обрывы соединения - Инструмент автоматически возобновляет загрузку
Технические детали
Схема загрузки

Получение списка файлов из HF репозитория с размерами и хэшами
Пользователь выбирает файлы для загрузки
Проверка места на диске - остановка если недостаточно
Параллельная загрузка с возобновлением (Range запросы)
Умная обработка Range fallback (200 vs 206)
Прогресс в реальном времени с расчетом скорости
Проверка SHA256 после загрузки
Склейка через llama-gguf-split
Расчет ETA
