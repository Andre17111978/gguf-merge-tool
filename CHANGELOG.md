# Changelog

Все заметные изменения в проекте "GGUF Merge Tool" будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
и проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

---

## [10.0] - 2026-07-07

### Добавлено
- **Загрузка локальных файлов** — новая кнопка "Локальные" в GUI, позволяющая выбрать папку с GGUF-файлами на диске и работать с ними без скачивания с Hugging Face.
- **История запросов** — в строке ввода репозитория теперь выпадающий список с последними 10 введенными ID репозиториев для быстрого доступа.

### Исправлено
- **Статусбары при проверке хэшей** — теперь корректно отображают прогресс во время SHA256-верификации.
- **Статусбары при склейке** — прогресс-бар теперь обновляется плавно и правильно отображает завершение процесса.

---

## [9.2.0] - 2024-07-07

### Добавлено
- **ETA (оставшееся время)** — показывает, сколько осталось ждать до завершения скачивания.
- **Отображение скорости в MB/s** в реальном времени.
- **Защита от Range Fallback** — если сервер не поддерживает докачку, файл не портится (интеллектуальная обработка HTTP 200 vs 206).
- **Полная документация на 6 языках** (Английский, Русский, Китайский, Испанский, Немецкий, Японский).

### Исправлено
- **HTTP 200 vs 206** — теперь правильно обрабатывается, когда сервер не поддерживает Range запросы.
- **Прогресс-бар при параллельной загрузке** — теперь обновляется корректно.
- **Умный поиск llama-gguf-split** — больше не использует `os.walk` (было медленно).

### Улучшено
- **Логирование** — теперь URL пишутся только в DEBUG режиме (не засоряют лог).
- **Производительность поиска** — кэширование результатов.

## [9.1.0] - 2024-07-06

### Добавлено
- **Расчет скорости в реальном времени**.
- **Форматирование ETA** (дни, часы, минуты, секунды).
- **Отображение прогресса** в формате: `[1/4] файл.gguf: 45% | 4.5 MB/s | ETA: 2ч 15м`.

## [9.0.0] - 2024-07-06

### Добавлено

- **Умное обнаружение llama-gguf-split**.
- **Параллельная загрузка** (ThreadPoolExecutor).
- **SHA256 проверка хэшей**.
- **Прокручиваемый список файлов** с поддержкой колеса мыши.
- **Сохранение пути к llama.cpp** между сессиями.
- **Кнопки "Выбрать всё" и "Скачать всё"**.
- **Информация о выбранных файлах** (количество, размер).

### Исправлено
- Производительность поиска llama-gguf-split (без `os.walk`).
- Обработка ошибок при скачивании.
